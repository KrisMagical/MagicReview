# Phase 8 GitHub App

Phase 8 lets ReviewAgent receive GitHub Pull Request webhooks, review the PR diff, and publish ReviewAgent findings back to the PR.

## Flow

Pull Request -> GitHub Webhook -> ReviewAgent GitHub App -> ReviewService.review_diff -> PR summary comment and inline comments.

## GitHub App Permissions

Repository permissions:

- Contents: Read
- Pull requests: Read & Write
- Issues: Read & Write
- Checks: Read & Write, optional
- Metadata: Read

Webhook events:

- Pull request

Recommended pull request actions are handled:

- opened
- synchronize
- reopened
- ready_for_review

Other actions are ignored.

## Environment Variables

```bash
GITHUB_APP_ID=12345
GITHUB_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----..."
GITHUB_WEBHOOK_SECRET="your-webhook-secret"
GITHUB_APP_NAME="ReviewAgent"

REVIEWAGENT_GITHUB_ENABLE_INLINE_COMMENTS=true
REVIEWAGENT_GITHUB_ENABLE_SUMMARY_COMMENT=true
REVIEWAGENT_GITHUB_ENABLE_AGENTS=false
REVIEWAGENT_GITHUB_ENABLE_LLM=false
REVIEWAGENT_GITHUB_CONFIG_PATH=
REVIEWAGENT_GITHUB_MAX_INLINE_COMMENTS=30
REVIEWAGENT_GITHUB_FAIL_ON=high
REVIEWAGENT_GITHUB_SAVE_RESULTS=true

REVIEWAGENT_GITHUB_HOST=0.0.0.0
REVIEWAGENT_GITHUB_PORT=8000
```

Do not commit the private key or webhook secret.

## Start The Server

```bash
python -m reviewagent.integrations.github.app
```

After installing the package:

```bash
reviewagent-github-app
```

The server exposes:

- `GET /health`
- `POST /webhook`

## Local Development

Use a tunnel such as ngrok and set the GitHub App webhook URL to:

```text
https://your-tunnel.example.com/webhook
```

Set the same webhook secret in GitHub and `GITHUB_WEBHOOK_SECRET`.

## Summary Comment

ReviewAgent upserts a single summary comment using this marker:

```html
<!-- reviewagent-summary -->
```

Example:

```markdown
## ReviewAgent Report

Summary:
- Critical: 0
- High: 2
- Medium: 4
- Low: 8

Top Issues:
1. [HIGH] SQLInjectionRule - app/db.py:20

Generated by ReviewAgent.
```

## Inline Comments

Inline comments are only created for issues that map to added or modified PR diff lines. Each comment contains a stable marker:

```html
<!-- reviewagent-inline:SQLInjectionRule:app/db.py:20 -->
```

This avoids posting the same inline comment repeatedly.

## Checks

ReviewAgent can create a simple `ReviewAgent` check run when `REVIEWAGENT_GITHUB_FAIL_ON` is set. The conclusion is `failure` if issues at that severity or higher exist, otherwise `success`.

## Dashboard Persistence

When `REVIEWAGENT_GITHUB_SAVE_RESULTS=true`, PR review results are saved to the Dashboard SQLite database. ReviewAgent stores issue summaries and sanitized metadata such as action, sender, author, base sha, and head sha. It does not store tokens, private keys, or full diff text.

## Security

- Webhook signatures are verified with `X-Hub-Signature-256`.
- PR code is not executed or imported.
- ReviewAgent uses static diff analysis by default.
- LLM review is disabled by default.
- Inline comments are capped by `REVIEWAGENT_GITHUB_MAX_INLINE_COMMENTS`.
- GitHub API errors are caught and returned as safe webhook responses.

## Current Limits

- The default PR workflow reviews diffs, not full checked-out repositories.
- Enterprise project rules are limited when only diff text is available.
- Dashboard persistence is optional and local SQLite based.
- Phase 9+ can add richer dashboard workflows and deployment hardening.
